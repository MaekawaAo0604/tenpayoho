# Twitter Developer Portal 申請用テンプレート

Twitter API申請時に使用する英文テンプレートです。コピー&ペーストして使用してください。

---

## 📝 Application Form（申請フォーム）

### 1. What's your use case? / Use Case
**選択**: `Making a bot`

---

### 2. Describe all of your use cases of X's data and API

以下の英文をコピーして貼り付けてください：

```
I am developing an automated weather information bot that posts daily hair condition forecasts for people with curly hair in Japan.

The bot will:
- Generate a daily weather map image showing humidity, dew point, and precipitation data for 6 major Japanese cities (Sapporo, Sendai, Tokyo, Nagoya, Osaka, Fukuoka)
- Post this image once per day (around 7:30 AM JST) with descriptive text and relevant hashtags
- Help people with curly hair plan their hairstyle and hair care based on weather conditions

I will use the Twitter API for:
- Posting tweets with media (v2 Tweet endpoint)
- Uploading weather map images (v1.1 Media upload endpoint)

Data usage:
- I will NOT store or analyze any Twitter user data
- I will NOT collect, process, or share any tweets or user information
- I will only POST content (weather forecasts) to my own account
- The bot operates fully automated on a daily schedule

This is a non-commercial, educational project aimed at providing helpful weather information to the curly hair community in Japan.
```

---

## 🎯 申請のポイント

### ✅ DO（推奨）
- 明確で具体的な説明
- 天気情報を提供する教育的・公益的な目的を強調
- データ収集や分析を**しない**ことを明記
- 投稿のみ（Read機能を使わない）であることを強調
- 1日1回という低頻度を明記

### ❌ DON'T（避ける）
- 曖昧な説明
- 商用利用の示唆
- データ収集・分析の可能性
- スパム的な大量投稿の示唆
- 他ユーザーとのインタラクション

---

## ⏱️ 審査期間

- **Free/Basic**: 即座に承認されることが多い
- **Elevated**: 数時間～数日（通常24時間以内）

---

## 📧 承認後の対応

承認されたら以下を確認：
1. App permissions が **Read and Write** になっているか
2. API Key, API Secret を保存
3. Access Token, Access Secret を生成・保存
4. GitHub Secrets に登録

詳細は [TWITTER_API_SETUP.md](TWITTER_API_SETUP.md) を参照してください。

---

## 🆘 却下された場合

稀に却下されることがあります。その場合：

1. より詳しい説明を追加
2. 公益性・教育目的を強調
3. データ保護への配慮を明記
4. サンプル画像やGitHubリポジトリのURLを提供

再申請は可能です！
